Yazılım Kalite Güvence Testi Dersi için aşağıdaki kurallara göre işlemler yapılacaktır. 
Öncelikle eklenecek tüm işlemler bu kurallardan geçirilecek ve bu kurallara uygun olacak şekilde yapılacaktır.
Kurallar 
1-	Bu bir yazılım testi üretme ve test etme projesidir. 
2-	Proje de C# kodları kullanılacaktır.
3-	Veri tabanı olarak SQLite kullanılacaktır.
4-	Api Dokumantasyonu olarak Swagger UI kullanılacaktır.
5-	Github ta yer alan Readme.md dosyası şunlar yer alacaktır. Rules klasörü içerisinde yer alan md dosyaları asla Readme.md dosyası içerisinde yer almayacaktır. Kesinlikle bunlar dışına çıkılmayacak. Bu başlıklar alanı belirlenecek ve uygulama güncellendikçe buralar yazılacaktır
-	Proje açıklaması ve kullanılan teknolojiler
-	Kurulum talimatları (adım adım)
-	API endpointlerinin listesi ve kullanım örnekleri
-	Swagger/OpenAPI dokümantasyonuna erişim URL’si (Example: http://localhost:3000/api-docs)
-	Test çalıştırma komutları
-	CI/CD badge’leri (GitHub Actions, Codecov)
6-	Uygulama da test yapısı olacak
7-	Uygulamayı yapay zekanın yazdığı belli olmayacak. Kullanıcı yazmış gibi davranılacaktır
